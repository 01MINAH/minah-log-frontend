# 대기만성 (Late Bloomer)

**소프트웨어학과 경진대회**에서 우수한 성적을 거둔 '자격증을 준비 중인 학습자들을 위한 효율적인 기출문제 학습 및 추천 사이트'입니다.

## 프로젝트 소개
정보처리기사 등 국가기술자격증 필기 시험을 준비하는 독학 수험생을 타겟으로 한 오답 분석, 최신 기출문제 모의고사, OCR 기반 문제 자동 변환, AI 기반 취약점 파악 및 학습 강좌 자동 매칭 웹 플랫폼입니다.

## 주요 특징 및 기능
- **OCR 문제지 스캔**: 기출문제 PDF나 캡처본을 이미지 업로드하면 OCR로 글자를 판독하여 디지털 시험 문제로 자동 빌드.
- **모의고사 타이머 및 자동 채점**: 실제 시험장과 동일한 환경의 타이머 제공, 채점 후 자동 오답 노트 생성.
- **데이터 분석 대시보드**: 자격증 과목별 정답률 및 취약 영역 분석.
- **10,000건 이상의 기출문제 아카이브**: 데이터 크롤링을 통해 구축된 신뢰성 높은 DB 에셋.

## 사용 기술
- **Frontend**: HTML5, Vanilla JavaScript, CSS Grid
- **Backend**: Python, Django, MySQL
- **Libraries**: Tesseract OCR, BeautifulSoup (Web Crawling)

---

## 내가 기여한 부분
- **OCR 문자 인식 파이프라인**: Python OCR 패키지를 가공해 이미지에서 텍스트와 보기를 정확히 파싱하여 DB 레코드로 적재하는 모듈 빌드.
- **대량의 기출 데이터 크롤링**: 자격증 커뮤니티 및 공인 사이트에서 10,000건 이상의 문제와 정답지 데이터를 크롤러(BeautifulSoup)로 자동 수집 및 정제.
- **DB 환경 구축 및 최적화**: 도메인별 테이블 분리 설계 및 ERD 다이어그램 작성 (기여도 85+%).

## 트러블슈팅 및 극복 과정

### 문제: 쿼리 풀스캔으로 인한 동시 채점 시 서버 응답 지연
- **현상**: 여러 사용자가 동시에 모의고사를 제출하고 전체 문항 오답 분석을 시도할 때, 대량의 문제 아카이브 테이블에서 조인 쿼리 풀스캔이 발생하여 데이터베이스 CPU 점유율이 90% 이상으로 치솟고 응답 속도가 약 4초가 소요되는 문제가 있었습니다.
- **해결**: 데이터베이스 쿼리 실행 계획(`EXPLAIN`)을 분석하여 오답 테이블과 문제 테이블의 외래키 조건 및 풀스캔 구간에 **복합 인덱스(Composite Index)**를 부여했습니다. 또한, 자주 조회되는 기출 문항은 캐싱 레이어를 통해 메모리에 올려 둠으로써 기존 풀스캔 쿼리 문제를 완벽히 방지하여 서버 평균 응답 시간을 **약 30.34% 단축(4.2초 -> 2.9초)** 시켰습니다.
