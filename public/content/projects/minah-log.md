# MIN-AH LOG

현재 개발 중인 개발자 포트폴리오 겸 기술 블로그 프로젝트입니다.

프로젝트 경험과 기술 학습 과정을 기록하고 공유하며, 개발자로서의 성장 과정을 담기 위한 공간으로 기획했습니다.

---

## 프로젝트 소개

포트폴리오와 기술 블로그를 하나의 웹 애플리케이션으로 구현한 프로젝트입니다.

단순히 프로젝트를 나열하는 형태를 넘어 **방문자가 직접 프로젝트를 탐색하고 기술 문서를 읽으며 개발 과정을 경험할 수 있는 포트폴리오**를 목표로 설계했습니다.

---

## 개발 과정

### Vibe Coding 도입

이번 프로젝트에서는 기존의 코드 중심 개발 방식에서 벗어나 **Antigravity IDE**를 활용한 Vibe Coding 방식을 처음 적용했습니다.

기능 요구사항을 먼저 정리한 뒤 AI와 함께 컴포넌트 구조와 UI를 설계하고, 생성된 코드는 직접 검토하고 수정하며 개발을 진행했습니다. 이를 통해 반복적인 구현 시간을 줄이고 서비스 구조와 사용자 경험 개선에 더욱 집중할 수 있었습니다.

### Antigravity IDE를 선택한 이유

이번 프로젝트에서는 무료로 사용할 수 있는 AI 기반 IDE를 찾던 중 Antigravity IDE를 선택했습니다.

단순히 AI가 생성한 코드를 사용하는 데 그치지 않고, 컴포넌트 구조를 함께 설계하고 생성된 코드를 직접 검토·수정하는 과정을 반복하며 AI 기반 개발 워크플로우를 경험하고자 했습니다.

이를 통해 반복적인 구현 시간을 줄이고 서비스 구조와 사용자 경험을 개선하는 데 더욱 집중할 수 있었습니다.

### Jira 활용
![Jira 활용](/assets/minah_log/jira_image.png)

프로젝트를 진행하면서 기능을 단위별 Task로 관리하고 개발 진행 상황을 추적하기 위해 Jira를 활용했습니다.

또한 **이슈 번호 기반 브랜치 전략**을 적용하여 각 기능을 독립적으로 개발했습니다.

- 기능(Task) 생성 및 일정 관리
- 개발 진행 상황 추적
- 이슈 번호 기반 브랜치 생성

### GitHub Desktop 활용

GitHub Desktop을 활용하여 브랜치 생성, 변경 사항 확인, 커밋 및 병합 작업을 진행했습니다.

## 트러블슈팅

### 폰트 스타일의 일관성과 가독성 부족

#### 문제

초기에는 `Jua` 폰트를 전체에 적용했지만, 블로그 본문과 포트폴리오처럼 텍스트가 많은 화면에서는 가독성이 떨어지고 전문적인 느낌이 부족했습니다.

#### 해결

- Heading : `Plus Jakarta Sans`
- Body : `Noto Sans KR`

헤딩과 본문에 각각 적합한 폰트를 적용하여 영문과 국문 모두 높은 가독성을 확보하고, 전체적인 디자인의 완성도를 높였습니다.
