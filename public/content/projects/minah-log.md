# MIN-AH LOG

현재 진행 중인 개발자 포트폴리오 겸 기술 블로그 프로젝트입니다. 이 사이트는 개발자로서의 나 자신을 표현하고, 배운 지식을 기록하며, 다른 사람들과 소통하기 위한 공간입니다.

## 프로젝트 소개
포트폴리오와 블로그를 하나로 결합한 웹 애플리케이션으로, 단순한 이력서 형식이 아닌 방문자가 직접 조작하고 경험할 수 있는 완성도 높은 포트폴리오를 제공하는 것을 목표로 설계되었습니다.

## 주요 특징 및 기능
- **포트폴리오 & 블로그 통합**: 프로젝트 상세 이력뿐만 아니라 개발 블로그 글을 함께 게시하여 나의 기술 성장을 보여줍니다.
- **반응형 디자인**: 다양한 기기(PC, 태블릿, 모바일)에 최적화된 화면을 제공합니다.
- **테마 시스템**: 눈의 피로를 덜어주는 라이트 모드 및 다크 모드 토글 기능을 지원합니다.
- **동적 마크다운 로딩**: 마크다운 파일(`.md`)을 읽어와서 풍부하고 유연한 형식의 글을 표현합니다.

## 사용 기술
- **Frontend**: React, React Router Dom, CSS Variables
- **Icons**: Lucide React
- **Markdown Parsing**: Marked JS
- **Design System**: Warm Pastel Beige

## 내가 기여한 부분
- **디자인 기획 및 레이아웃 설계**: 전체적인 그리드 구조와 파스텔 베이지 톤의 테마 기획
- **컴포넌트 컴포지션**: 재사용 가능한 Layout, Header, Footer, Markdown 렌더러 구현
- **마크다운 렌더링 파이프라인**: 서버 API 없이 static 환경에서 동적으로 `.md` 파일 정보를 fetch하여 안전하게 클라이언트에 파싱하는 시스템 구축

## 트러블슈팅 및 극복 과정

### 문제: 폰트 스타일의 일관성 및 전문성 결여
- **현상**: 초기에는 `'Jua'` 폰트가 전체적으로 적용되어 있었는데, 텍스트가 많은 블로그 본문이나 정형화된 포트폴리오 내용에서는 가독성이 크게 떨어지고 전문적인 인상을 주지 못하는 문제가 있었습니다.
- **해결**: 영문과 숫자의 정교함을 위해 `Plus Jakarta Sans`를 헤딩에 적용하고, 국문 폰트로 고해상도 가독성을 제공하는 `Noto Sans KR`을 혼합 구성하여 고급스럽고 정돈된 가독성을 확보했습니다.

### 문제: static 호스팅에서의 dynamic markdown 렌더링
- **현상**: SPA(React) 빌드 시, 별도의 백엔드 없이 마크다운 파일을 로드해야 했습니다.
- **해결**: 마크다운 파일을 `public/content/` 폴더 하위에 정적 에셋으로 관리하고, React component 마운트 시 `fetch` API를 사용하여 비동기로 데이터를 들고 와 클라이언트 단에서 파싱하도록 설계했습니다. 이를 통해 빌드 속도에 영향을 미치지 않고 콘텐츠를 자유롭게 확장할 수 있는 파이프라인을 구축했습니다.
