# PopFlix (팝플릭스)

**LG유플러스 유레카 SW 교육과정**의 경진대회 '융합 프로젝트' 최우수작으로 선정된 영화 리뷰 및 맞춤 소셜 서비스입니다.

## 프로젝트 소개
다양한 영화 정보 API를 가공하여, 사용자들이 실시간으로 평점과 한 줄 리뷰를 남길 수 있고, 개인별 시청 취향에 맞춘 영화 추천 시스템 및 다른 사용자들과 소셜 소통을 나눌 수 있는 융합 미디어 리뷰 솔루션입니다.

## 주요 특징 및 기능
- **영화 상세 조회 및 별점**: 영화 상세 정보 및 풍부한 메타데이터 제공, 유저 평가 평점 반영.
- **실시간 소셜 피드**: 영화 마니아들의 한 줄 평 및 세부 리뷰를 한눈에 볼 수 있는 소셜 타임라인.
- **취향 맞춤 필터링**: 선호하는 장르, 감독, 배우 기반의 개인화 추천 엔진.
- **CI/CD 파이프라인**: 배포 속도 향상 및 자동화를 위한 CI/CD 아키텍처.

## 사용 기술
- **Frontend**: React, React Router Dom, Context API, CSS (Styled components 느낌의 컴포넌트화)
- **Backend**: Spring Boot, JPA, MySQL, AWS RDS
- **Infrastructure**: AWS EC2, GitHub Actions, Docker

---

## 내가 기여한 부분
- **프론트엔드 핵심 컴포넌트 개발**: 영화 상세 목록 필터링, 평점 컴포넌트 및 피드 UI 제작.
- **Notion 연동 및 상세 기획서 관리**: 디자이너와 백엔드 개발자 간의 원활한 소통을 위한 데이터 구조 명세 정립.
- **ERD 다이어그램 공동 설계**: MySQL 릴레이션 다이어그램 설계 참여, 1:N 및 M:N 매핑 테이블 최적화.

## 트러블슈팅 및 극복 과정

### 문제: 동시 접속 시 영화 검색 및 필터링 속도 저하
- **현상**: 많은 영화 데이터를 복합 장르와 평점 조건으로 조회할 때 대기 시간이 급증하는 병목 현상이 발생했습니다.
- **해결**: 데이터베이스에 다중 인덱스(Composite Index)를 설정하고, 프론트엔드에서는 빈번한 API 호출을 막기 위해 **Debounce** 기법을 적용한 검색 인풋을 개발하여 서버 호출량을 약 60% 절감하고 응답 속도를 200ms 이하로 유지했습니다.
